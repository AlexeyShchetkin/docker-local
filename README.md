# Docker environment for local development

Модульная Docker-инфраструктура для разработки PHP приложений.

## Особенности

- **Модульность**: Отдельные docker-compose файлы для каждого сервиса (`app`, `web`, `pgsql`, `mysql`, `redis`, `horizon`, `scheduler`).
- **Удобство управления**: Использование `Makefile` для повседневных задач.
- **Изоляция**: Настраиваемая подсеть для каждого проекта.
- **Готовность к Laravel**: Включены конфигурации для Horizon и планировщика задач.

## Требования

- Docker >= 20.10
- Docker Compose v2
- Make

## Быстрый старт

1. **Инициализация конфига**:
   ```bash
   make init
   ```
   Это создаст файл `.docker.local` на основе примера.

2. **Настройка параметров**:
   Отредактируйте `.docker.local`, установив уникальный `PROJECT_ALIAS` и `NETWORK_SUBNET`.

3. **Активация модулей**:
   По умолчанию в корне находятся только `app.yml` и `web.yml`. Чтобы добавить другие сервисы (БД, Redis и т.д.), скопируйте нужные файлы из папки `modules/` в корень проекта:
   ```bash
   cp modules/pgsql.yml .
   cp modules/redis.yml .
   ```
   Для **MySQL/MariaDB** (вместо или в дополнение к PostgreSQL — не подключайте оба к одному приложению без явной необходимости):
   ```bash
   cp modules/mysql.yml .
   ```
   В образе приложения уже собраны расширения **pdo_mysql** и **mysqli**; в `.docker.local` используйте те же `DB_NAME`, `DB_USERNAME`, `DEFAULT_PASSWORD`. Хост БД из контейнера приложения: IP `mysql` в compose-сети или `${NETWORK_SUBNET}.33` (см. `modules/mysql.yml`).

   *Все `.yml` файлы в корне будут автоматически подключены при запуске.*

   Команды `make build`, `make start` и остальные, вызывающие Docker Compose, требуют наличия файла `.docker.local` и непустого `PROJECT_ALIAS` (после `make init` отредактируйте значения в `.docker.local`).

4. **Сборка и запуск**:
   ```bash
   make build
   make start
   ```

5. **Размещение кода**:
   Поместите ваш исходный код в директорию `src/`. Он будет автоматически смонтирован в `/var/www/html/` внутри контейнеров.

## Доступные команды (Make)

| Команда | Описание |
| :--- | :--- |
| `make init` | Создать локальный конфиг из шаблона |
| `make build` | Собрать Docker-образы |
| `make start` | Запустить все сервисы |
| `make stop` | Остановить и удалить контейнеры |
| `make restart`| Перезапустить сервисы |
| `make shell` | Вход в консоль контейнера `app` |
| `make logs` | Просмотр логов контейнера `app` |
| `make artisan`| Выполнить artisan команду (пример: `make artisan migrate`) |
| `make help` | Справка по командам |
| `make check-config` | Проверить `.docker.local` и `PROJECT_ALIAS` |

## Структура проекта

- `.docker/local/`: Конфигурационные файлы (Dockerfile, php.ini, nginx.conf).
- `modules/`: Дополнительные модули (база данных, очереди и т.д.).
- `src/`: Директория для исходного кода приложения.
- `*.yml`: Активные конфигурации Docker Compose (в корне).
- `Makefile`: Точка входа для управления инфраструктурой.
- `.docker.local`: Локальные переменные окружения (не коммитится).

## Настройка сети и доступа

После запуска (`make start`) проект будет доступен по IP-адресу, указанному в выводе команды. По умолчанию (при `NETWORK_SUBNET=10.10.100`):
- Web-сервер: `http://10.10.100.4`
- PostgreSQL (если подключён `pgsql.yml`): `10.10.100.3`, порт `5432`
- MySQL/MariaDB (если подключён `mysql.yml`): `10.10.100.33`, порт `3306`
